## The Karpathy Decomposition Prompt - make Claude think before answering

### Feature Idea

Use when you're facing a complex idea, product, or technical problem.

Prompt:

"Approach this like a systems thinker.

1. Define the core problem clearly
2. Identify assumptions
3. List constraints and unknowns
4. Break into sub-problems
5. Propose 3 different approaches
6. Compare tradeoffs
7. Choose best approach
8. Give step-by-step execution
9. Highlight failure points
10. Suggest improvements after v1

Problem: [paste]"

This forces structured reasoning instead of shallow answers.

## First Principles Builder - deep understanding instead of summaries

Prompt:

"Explain this from first principles.

1. Start with the most fundamental concepts.
2. Build upward layer by layer.
3. Avoid analogies initially.
4. Define terminology clearly.
5. Show how each layer connects.
6. Then provide mental model.
7. Then show real-world application.
8. Then give common misconceptions.
9. Then summarize in 5 bullet points.

Topic: [insert]"

5. what's failing
6. gaps in the market
7. emerging trends
8. contrarian insights

10. actionable takeaways

4. data flow
5. tech stack

7. edge cases
8. scaling strategy
9. possible bottlenecks
10. v2 improvements

## Meta Prompt Optimizer - upgrade every prompt you write

Prompt:

"Rewrite this prompt to maximize output quality.

Improve:
1. clarity
2. structure
3. constraints
4. output format
5. reasoning depth
6. specificity

Return:

1. optimized prompt
2. why it's better
3. when to use

Prompt: [paste]"

This compounds prompt quality over time.

## Expert Mode Switch - force high-level answers

Prompt:

"Answer as a senior engineer explaining to another engineer.

1. Avoid beginner explanations.
2. Be concise but technical.
3. Focus on implementation.
4. Mention tradeoffs.
5. Include pitfalls.
6. Include best practices.

Question: [insert]" 

Claude shifts from teacher → practitioner

## Thinking Partner Prompt - challenge your ideas

Prompt:

"Act as a critical thinking partner.

1. Do not agree blindly.
2. Challenge assumptions.
3. Point out weak logic.
4. Suggest alternatives.
5. Identify risks.
6. Improve the idea.
7. Propose better direction.

Idea: [paste]"

This is where Claude becomes a real collaborator.
